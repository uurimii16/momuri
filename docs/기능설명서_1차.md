## 1. 서비스 개요

**momuri(머무리)** 는 데이터가 진단한 **"살릴 자원은 있는데 소외된 쇠퇴 소도시(숨은 매력지)"** 를
문화유산 매력으로 발견하게 하고, **하루→체류형 코스**로 현실화하며, **한국관광공사 TourAPI 콘텐츠**로 채우고,
**로컬 일자리·워케이션·장기 숙소**까지 연결해 **일회성 관광을 체류·재방문(생활인구 유입)** 으로 전환하는 체류형 로컬 관광 플래너입니다.

### 과제4 정합 — 한 흐름
```
문화유산 매력으로 유입   →   하루 코스로 경험   →   체류형(1박+·한달) 코스   →   로컬 일자리·워케이션·장기숙소
   (왜 가나 = 유입 훅)         (어떻게 도나)          (왜 머무나 = MVP 핵심)         (생활인구 = 정착)
```
- **유입 훅**: 문화유산 매력(데이터 엔진이 "왜 이 소도시인가"를 근거로 선정)
- **MVP 핵심**: 체류·재방문·로컬 일자리 매칭 → 과제4의 "생활인구 유입"에 정면 대응
- 단순 숨은명소 발굴에 그치면 과제2가 됨 → momuri는 **머무는 이유(체류·일자리)를 실제 기능으로** 구현

### 차별점 (moat)
경쟁작 다수가 "LLM 추천 플래너"로 *발견*에 그칠 때, momuri는
**① 쇠퇴지수** (어디가 활성화 우선지인가) + **② 동선 네트워크** (현실적 코스 + 소외 진단) 라는
**데이터 엔진**이 추천의 근거가 되고, 나아가 **로컬 일자리·체류로 관계인구까지** 잇습니다.

## 3. 화면별 기능 설명 (6개 페이지)

### 3-0. 랜딩 — `index.html`
- 서비스 소개 + **5개 기능 페이지 진입 카드**(숨은 매력지 / 하루 코스 / 체류 플래너[NEW] / 쇠퇴지수 지도 / 방법론).
- 상단 내비게이션·로고 클릭으로 어느 페이지서든 홈 복귀.

> [캡처 ①] 랜딩 페이지 — 진입 카드 5개가 보이도록 전체 화면.

---

### 3-1. 숨은 매력지 지도 — `momuri.html`  *(유입 훅)*
**"쇠퇴 × 문화유산" 사분면으로 선정된 숨은 매력지 12곳**을 지도에 표시합니다.
- 지역 마커 + **POI 핀 클릭 팝업**(사진·카테고리·주소 — 한국관광공사 TourAPI 콘텐츠).
- 마커 **색상 범례**(카테고리 구분).
- 각 지역 팝업의 **"🏡 며칠 머무르기(체류·일자리)"** 버튼 → 체류 플래너(`stay.html`)로 연결.

> [캡처 ②] 숨은 매력지 지도 — POI 핀 팝업(사진·주소)이 열린 상태.
>
> **대응 해시태그**: `#숨은_매력지` `#문화유산_로컬여행` `#데이터_기반_추천`

---

### 3-2. 하루 코스 타임라인 — `course.html`  *(경험)*
명소·맛집을 하루 리듬으로 묶은 **동선 타임라인**과 지도 실도로 경로선을 제공합니다.
- 번호 스톱(도착 시각·머무는 시간·사진) + 지도 경로선.
- **자동차 / 대중교통 토글** — 대중교통은 버스번호·환승·구간 소요분(ODsay) 표시, 경로 없는 구간은 "🚗 자동차 권장".
- 데이터: `courses.json`(12개 코스, 자동차 OSRM + 대중교통 ODsay).

> [캡처 ③] 하루 코스 — 타임라인 + 지도 경로선, 대중교통 토글이 켜진 상태.
>
> **대응 해시태그**: `#로컬관광`

---

### 3-3. 체류형 로컬 관광 플래너 — `stay.html`  *(★ 과제4 MVP 핵심)*
일회성 방문을 **체류·재방문**으로 전환하는 핵심 화면입니다. 왼쪽 패널 + 오른쪽 지도.
- **체류 조건 토글**: `1박2일` / `2박3일` / `한 달 살기` — 2030 워케이션·한달살기 타깃.
- **Day 탭** → 각 날짜별 타임라인(주간 스톱 + 구간 이동시간 + 🏠 야간 숙소) + 지도 경로/마커.
- **로컬 일자리 3단 패널**:
  1. **인력난 신호 지표** — 문화유산 수·방문 잠재·청년 유출 프록시(향후 KOSIS 구인배수로 정밀화).
  2. **실물 채용 딥링크** — "이 지역 채용 보기(고용24)" + "공공기관 채용(잡알리오)".
  3. **공공기관 채용 카드** — `jobs.json`(잡알리오, 기재부 공공기관 채용 OpenAPI)의 로컬·인근 공고.
- **장기 숙소 카드**(TourAPI 숙박) + **생활인구 스토리 박스**(쇠퇴지수·체류일수 → 생활인구 유입 서사).

> **일자리 3단 구조 = 데이터 접근 현실을 정직하게 우회한 설계**
> 민간 실시간 채용 API(워크넷 채용정보)는 기업회원 전용이라 개인 접근 불가 →
> 실물 중소기업 채용은 **딥링크**로, 공식 API로 되는 공공기관은 **카드**로 보완하고, 지역 인력난은 **지표**로 근거화.

> [캡처 ④] 체류 플래너 — 체류조건 토글 + Day 탭 타임라인 + 일자리 3단 패널이 한 화면에.
>
> **대응 해시태그**: `#체류형관광` `#한달살기` `#워케이션` `#2030청년` `#로컬일자리` `#생활인구`

---

### 3-4. 시군구 쇠퇴지수 단계구분도 — `decline-map.html`  *(데이터 엔진 = 근거)*
숨은 매력지 선정의 근거인 **복합쇠퇴지수**를 전국 시군구 단계구분도(Jenks 자연분류)로 시각화합니다.
- 지역 클릭 시 **방문·매력·연결 3축 분해 + "주된 쇠퇴 요인"** 표시 → 단순 순위가 아니라 *왜* 쇠퇴했는지를 지도가 증명.
- **쇠퇴 상위 Top15 랭킹**(클릭 시 해당 지역으로 flyTo), hover 하이라이트.
- GeoJSON 경량화(9.2MB→0.67MB)로 빠른 로딩.

> [캡처 ⑤] 쇠퇴지수 지도 — 한 지역 클릭해 3축 분해 패널 + Top15 랭킹이 보이는 상태.
>
> **대응 해시태그**: `#데이터_기반_추천` `#쇠퇴지_관광활성화`

---

### 3-5. 지표 방법론 — `methodology.html`
복합쇠퇴지수(방문·매력·연결 3축)의 산출 방법·데이터 출처를 투명하게 설명합니다.

## 5. 한국관광공사 OpenAPI(TourAPI) 활용 내역  *(필수 · 배점 20점)*

> ⚠️ 공사 OpenAPI 미충족 시 심사 제외. momuri는 콘텐츠 뼈대 전반에 **TourAPI 4.0 KorService2** 를 사용합니다.

| 용도 | 사용 오퍼레이션 | 반영 화면 |
|---|---|---|
| 지역 관광지·음식·숙박·체험 콘텐츠 | `areaBasedList2` / `locationBasedList2` / `detailIntro2` | momuri·course·stay |
| 숙박(장기 체류 숙소) | `locationBasedList2`(contentTypeId=32) | stay.html |
| 축제·행사("지금 갈 이유") | `searchFestival2` | momuri·course |
| 키워드·POI 검색 | `searchKeyword2` | 데이터 수집 |
| 지역코드 | `areaCode2` | 데이터 수집 |

- **키 관리**: `TOURAPI_KEY` 환경변수로만 사용(소스에 커밋 안 함).
- **축제 데이터 보정**: 쇠퇴 소도시는 등록 축제가 적고 차년도 일정이 늦게 반영됨 →
  최근 1년 창 조회로 연례 축제까지 포착하고 `status`(upcoming/annual)·`scope`(sigungu/sido) 태깅,
  시군구 0건이면 도(道) 단위 폴백. (예: 상주→경북 인근 연례 축제, 문경→문경사과축제)
- (보조) 방문자수 API(B551011, data.go.kr) = 쇠퇴지수 방문축 산출에 사용 — 공사 데이터.

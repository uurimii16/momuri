## 1. Phase 2에서 이번에 완성한 것 (2026-07-07)

### 1-1. 다일 체류 코스 — `analysis/build_courses.py` 확장
- 기존 "하루 5스톱 그리디 동선"을 **최대 3일**(`MAX_DAYS=3`)로 반복.
- **스톱 풀을 날짜 간 나눠 소비** → 날마다 다른 명소·맛집(중복 없음).
- **야간 = `stays[]`(TourAPI 숙박) 최근접 숙소** 자동 배정. 마지막 날은 체크아웃(야간 없음).
- **하위호환**: 최상위 `stops`/`legs`/`stay_total_min` = **Day1** 그대로 유지 → 기존 `course.html` 코드 **무수정** 동작.
- 데이터 재생성 완료: `map/data/courses.json` (12지역, 323KB). 대부분 3일·2박, 숙소 0곳 지역(문경·창녕)은 `night:null`.

**courses.json 새 스키마(핵심만)**
```
{ region, sido, sigungu, theme, center:[lng,lat],
  stops[], legs[],              // ← 하위호환(=Day1)
  car_total_min, stay_total_min, has_transit,
  n_days, n_nights, has_stay_data,
  days:[ { day,                 // 1,2,3
           stops:[{kind,name,addr,lat,lng,img,stay}],
           legs:[{car:{min,km,path[]}, transit}],
           night:{kind:"숙박",name,addr,lat,lng,img}|null,
           to_night:{car,transit}|null,   // 마지막 스톱→숙소 이동
           car_total_min, stay_total_min } ] }
```

### 1-2. `map/stay.html` — 과제4 핵심 화면 (신규)
왼쪽 패널 + 오른쪽 지도(routo-gl), 기존 테마·로고·출처표기 재사용.
- **체류 조건 토글**: `1박2일` / `2박3일` / `한 달 살기`(기본). 데모 데이터는 3일이 상한이라, 한달은 "대표 3일 + 이후 반복·정착" 프레이밍.
- **Day 탭** → 각 날 **타임라인**(주간 스톱 + 🚗구간분 + 🏠 야간 숙소) + **지도 경로/마커**.
- **로컬 일자리 3단 패널**:
  - ① **인력난 신호**: 문화유산 수·방문잠재·청년유출(연결성 음수 프록시) — 지표는 추후 KOSIS로 정밀화.
  - ② **채용 딥링크**: "이 지역 채용 보기(고용24)" + "공공기관 채용(잡알리오)".
  - ③ **공공기관 채용 카드**: `jobs.json`의 로컬 매칭 공고 + 인근 시·도 공고.
- **장기 숙소 카드**(momuri.json `stays[]`) + **생활인구 스토리 박스**(쇠퇴지수·체류일수·해시태그).

## 2. 로컬 일자리 3단 구조 — "공공기관만?" 방어 논리 (심사·PT용)

> 핵심 타깃은 **지방 중소기업 일자리**. 그런데 민간 채용 실시간 목록 API(워크넷 채용정보)는 **기업회원 전용**이라 개인 개발자 접근 불가(2026-07-07 확인). 그래서:

| 층 | 내용 | 데이터 | 해시태그 |
|---|---|---|---|
| ① 인력난 **지표** | "이 소도시는 일손이 부족" 근거 | KOSIS 구인배수 등(예정), 현재는 3축 프록시 | #생활인구 |
| ② 실물 채용 **딥링크**(메인) | 지방 중소기업 실물 채용으로 연결 | 고용24/워크넷 지역검색(키 불필요·항상 최신) | #로컬일자리 |
| ③ 공공기관 **카드**(보조·API) | 공식 오픈API로 신뢰성 있는 공고 | 잡알리오 `jobs.json` | #로컬일자리 |

**한 줄 방어**: "민간 실시간 채용 API는 개인 접근이 막혀 있어, 실물 중소기업 채용은 **딥링크**로 잇고, 공식 API로 되는 공공기관은 **카드**로 보완하며, 지역 인력난은 **지표**로 근거화했다. 데이터 접근성의 현실 한계를 정직하게 우회 → 실현가능성·성숙도."

## 3. 향후 작업 (우선순위 · 무엇을/왜/필요한 것)

### 🅰 지금 바로 (심사 데모 완성도)
1. **로컬 육안 검수 & 버그 픽스** — `stay.html` 실제로 열어 지도·타임라인·일자리·숙소가 다 뜨는지. (사용자가 확인 → 피드백)
2. **고용24 딥링크 키워드 필터 정밀화** — 현재 `keyword=시군구`로 landing. 실제 지역 필터가 걸리는 정식 파라미터/지역코드로 교체. (②층 신뢰도 ↑)
3. **숙소 0곳 지역 반경 폴백** — 문경·창녕은 `night:null`. `tourapi_fetch.py`에 `locationBasedList2`(지역 중심 반경 ~20km) 추가해 인근 숙소를 끌어와 야간 스톱 채우기.
4. **축제 라벨 프론트 반영** — momuri/course에서 `status`(annual→"연례(올해 일정 확인)")·`scope`(sido→"인근 축제") 라벨 표시. (데이터는 이미 태깅됨)

### 🅱 중기 (기획 깊이)
5. **인력난 지표 `job_index.json`** — KOSIS/고용통계에서 지역 **구인배수·인력부족률·청년 순유출**. → ①층을 실데이터로. (필요 시 KOSIS 오픈API 키 발급 안내)
6. **1차 제출용 "기능설명서" 작성** — 해시태그 ↔ 기능 매핑표(아래 §4)를 근거로 각 기능 캡처+설명. 공모전 1차 심사 제출물.
7. **로고 다듬기** — 현재 임시 SVG(핀+집).

### 🅲 확장 (Phase 3, 기능심사~)
8. **미식 지수/추천** — 문화유산+미식 소도시 랭킹.
9. **인바운드/외국인 체류** — 다국어·인바운드 콘텐츠.

## 5. 데이터 재생성 명령 모음 (원자료 바뀔 때만)

> 산출물(`map/data/*.json`)은 이미 커밋돼 있어, 웹만 볼 거면 재생성 불필요.
> Git Bash 기준. 윈도우 PowerShell은 `KEY=x cmd` 대신 `$env:KEY='x'; cmd`.

```bash
cd /d/코딩작업/tourgo-local-relation

# 관광 콘텐츠(관광지·음식·숙박·축제) → momuri.json
TOURAPI_KEY='발급키' python analysis/build_service_data.py 12

# 다일 체류 코스(자동차 OSRM) → courses.json   (대중교통까지면 ODSAY_KEY 추가)
python analysis/build_courses.py
ODSAY_KEY='키' ODSAY_REFERER='https://momuri.netlify.app' python analysis/build_courses.py

# 공공기관 채용(잡알리오) → jobs.json
TOURAPI_KEY='발급키' python analysis/build_jobs.py
```
- **API 키는 절대 커밋 금지** — 환경변수로만. data.go.kr 키 하나로 TourAPI·잡알리오 공용.
- ODsay 키는 공개 저장소라 커밋 안 함(없어도 자동차 데이터로 정상 작동). 발급처 lab.odsay.com.

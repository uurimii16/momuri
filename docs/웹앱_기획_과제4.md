# 웹·앱 부문 기획 — 과제4 (쇠퇴 소도시 관광 활성화 플래너)

> 2026 관광데이터 활용 공모전 **②-2 웹·앱 구현 부문 / 지정과제 4번** 출품 기획.
> 해결과제: *"단순 흥미 위주 일회성 관광의 한계 및 인구 감소 지역의 지속가능한 생활인구 유입 대책 부족"*
> 포스터 부문(`tourgo-tourism-decline`)의 분석 엔진(쇠퇴지수·동선 네트워크)을 **토대(moat)**로 재활용 → 한 데이터로 두 부문.
> (개념 확정: 2026-07-06)

---

## 0. 한 줄 정의

> **테마 매력으로 "숨은 로컬 매력지(쇠퇴 소도시)"를 매칭하고, 동선으로 현실적 코스를 만들어, 공사 관광데이터로 채우고, 체류·재방문으로 관계인구까지 잇는 관광 활성화 플래너.**

**관광 활성화가 메인.** 일자리·체류·생활인구는 관광의 *종착 심화 단계*(메인 아님).

## 1. 전략 — 왜 과제4 / 왜 관광이 주인공

- 이 부문 결과물 = **작동하는 웹/앱 서비스**(분석·보고서·정책제안 아님). 심사도 "완성 서비스"만 대상.
- 관광공사 대회 → **관광 활성화가 주인공**이어야. 일자리·생활인구는 "일회성 관광 → 체류·반복 관광"의 결과로 배치.
- 과제4 자체가 *"일회성 관광의 한계"* → 관광이 주인공인 게 오히려 과제에 충실.

## 2. 서비스 개념

### 2.1 매칭 로직 (매력 먼저, 동선은 거들기)
```
사용자 관심(테마: 미식/문화유산/한류/자연)
  → 그 테마를 진짜 충족하는 '숨은 매력 쇠퇴지' 매칭        ★매력 우선
  → 동선 네트워크로 현실적 코스화(며칠·경유·이동)          동선 보조
  → 공사 TourAPI로 콘텐츠(명소·먹거리·축제·숙박) 채움       필수 데이터
  → 체류·재방문으로 심화(+ 일자리·워케이션 매칭)            과제4 앵커
```
> ⚠️ "경유지라서 끼운다"는 논리는 약함 — **갈 이유(매력)가 없으면 안 감.** 그래서 매력이 1순위, 동선은 코스 현실화용.

### 2.2 3층 구조
| 층 | 내용 | 노출 |
|---|---|---|
| **① 토대(moat)** | 쇠퇴지수로 활성화 우선 소도시 선정 + 테마 유형 태깅 + 동선 네트워크(어느 허브에서 어떻게 연결/소외되나) | 오프라인 분석 → 데이터로 앱에 내장(사용자는 근거로만) |
| **② 서비스(★심사 main★)** | 관심 입력 → 숨은 매력지 코스 제시 → 공사 API 콘텐츠 → 체류·재방문·일자리 매칭 | 사용자가 쓰는 앱 |
| **③ 임팩트(스토리)** | 소외지 방문↑=관광 활성화 / 반복·체류=생활인구 / 일자리 연결=지방기업 인력난 → 청년↔지역 3면 | 기획력·발전성 점수 |

## 3. "숨은 로컬 매력지" 정의

**쇠퇴 + 테마자산 있음 + 소외됨(안 알려짐).**
- 자산 없는 극단 쇠퇴지(예: 울릉 — 멀고 자산 적음)는 **안 밀어냄**(정직한 필터).
- 자산은 있는데 동선에서 소외된 곳(문화유산형 타깃 등)만 → 사용자에겐 "진짜 가고 싶은 곳", 지역엔 "부당하게 소외된 곳".
- 포스터의 **타깃 유형화(문화유산형/자연힐링형)**와 직결.

## 4. moat — 남들과 다른 이유
1. **쇠퇴지수**: 어디가 활성화 우선지인가 (남들: 없음)
2. **동선 네트워크**: 매력지를 현실적 코스로 + 소외 진단 (남들: 데이터 없음)
3. **테마 유형화 + 미식·문화유산·한류 논거**: 지역별 관광 특화 (포스터 자산)
→ 경쟁작 다수는 "LLM 추천 플래너". 우리는 데이터 엔진이 추천의 근거.

## 5. 데이터 / API

### ⚠️ 대전제: 한국관광공사 OpenAPI **필수**(미충족 시 심사 제외, 배점 20점)
- ✅ **키 발급·활성화 완료**(2026-07-06): 국문 관광정보 TourAPI 4.0 `KorService2`. 행사정보(`searchFestival2`)는 이 서비스에 **포함**(별도 신청 불필요).

### 5.1 공사 OpenAPI = 콘텐츠 뼈대 (필수)
| 용도 | 오퍼레이션 |
|---|---|
| 지역 관광지·음식·숙박·체험 | `areaBasedList2` / `locationBasedList2` / `detailIntro2` |
| "지금 갈 이유" 축제·행사 | `searchFestival2` |
| 키워드·POI 검색 | `searchKeyword2` |
| 지역코드 | `areaCode2` |
| 인바운드 다국어 | 다국어 서비스(Eng/Chs/Jpn) — 지도 KO/EN/ZH/JA 완비 |

### 5.2 우리 엔진 (moat, 오프라인 분석 → 데이터 내장)
- 복합쇠퇴지수·활성화 타깃, 동선 네트워크 중심성, 재방문·추천의향, 테마 유형.
- 방문자수 API(B551011, data.go.kr 15101972) = 진단 방문축(이미 파이프라인 보유, 공사 API).

### 5.3 외부 API (선택, 공고상 자유 허용)
| 레이어 | 소스 |
|---|---|
| 일자리·워케이션(체류 심화) | 워크넷(고용24) 채용 API, 고용부 구인배수·인력부족률, KOSIS 청년고용 |
| 문화유산 콘텐츠 | 국가유산청 검색 API(좌표·나레이션 4개국어) |
| 타깃 정합 | 행안부 생활인구·인구감소지역 89 |
| 경로 | Routo(지도), 대중교통(ODsay/TAGO) |
| 내러티브 | LLM(테마 큐레이션·다국어) |

## 6. 화면 (기존 `map/` 재활용)
- 지도 셸·다국어·상세패널·루트 렌더 그대로. 탭에 **"관광 활성화 코스"** 추가.
- 좌: 관심(테마) 입력 → 숨은 매력지 카드 → 코스(스톱·이동). 우: 지도에 코스 오버레이.
- 상세: 쇠퇴/매력 근거 + 공사 API 콘텐츠 + (심화)체류·일자리.

## 7. 심사기준 매핑
**1차(100)**: 구현성 30(작동 서비스 필수) / 기획력 30(moat·매칭로직 독창성) / **공사 API 20**(5.1) / 발전성 20(로드맵·일자리·인바운드 확장).
**PT(100)**: 적정성 30 / 완성도 30 / 실용성 25 / 발표 15.

## 8. 일정·자격·유의
- **접수 2026.7.6~7.21 16:00**(약 2주, 신청+기획). 개발 7말~9월 / 기능심사 10월 / PT 10월 / 시상 11월.
- 팀당 최대 5명, 국내 거주자. **한국관광콘텐츠랩(api.visitkorea.or.kr) 회원가입 필수**.
- 중복 제한: ②-1 프롬프톤 제출작·① 부문과 동일 서비스 불가. 포스터는 별개 트랙(서비스 아님)이라 재활용 무방 가능성 높음 → 포스터 공모전 성격·수상 여부만 확인.

## 9. 확장 로드맵
- **Phase 1**: 숨은 매력지 매칭 + 테마 코스 + 공사 API 콘텐츠 + 지도. (작동 데모 성립)
- **Phase 2**: 동선 코스 현실화(경유·소요), 축제 시의성, 체류형 코스.
- **Phase 3**: 일자리·워케이션 매칭, 인바운드 다국어, 활성화 효과 대시보드(방문자수 API).

## 10. 과제4 앵커 (과제2와의 차이)
- "숨은 명소 발굴"로 끝나면 **과제2**가 됨. 우리는 그 발견을 **체류·재방문으로 이어 관계인구/생활인구**까지 → 과제4.
- 차별 = "한 번 발굴"이 아니라 "반복해서 머무는 관계".

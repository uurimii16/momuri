### 🅰 momuri 지도 핀을 tourgo식 반투명 물방울 핀으로 통일 (커밋 `f90759b`)
- 배경: 7/14에 course·stay는 tourgo식 CSS 물방울 핀(`.drop-pin`)으로 바꿨는데 **momuri(숨은 매력지)만 옛 불투명 SVG 핀**(`pinSVG`, `.mk-pin` opacity .72 + 흰 속점)으로 남아 있었음.
- 수정: `pinSVG()` **함수 본문 한 곳**만 CSS 물방울로 교체 → 호출부(gem 24px·POI 20px·범례 14px)가 **일괄 통일**.
  ```js
  // 지도 핀: tourgo식 반투명 물방울(CSS) — course·stay와 통일
  function pinSVG(color, w){
    return `<span class="drop-pin" style="--cat:${color};--sz:${w}px"></span>`;
  }
  ```
- CSS: `.mk-pin` 관련 규칙 → `.drop-pin`으로 교체. 반투명 radial-gradient + 흰 테두리 + `rotate(-45deg)` teardrop. 크기 `--sz` 변수화, 팁이 좌표를 가리키도록 하단 여백 `calc(var(--sz)*0.2)`.
- 색상: 매력지=청록 `#14b8a6` · 관광지=하늘 `#38bdf8` · 맛집=분홍 `#f472b6`(course와 동색). 문화유산 🏛 아이콘(`heriIconSVG`)은 카테고리 구분 위해 유지.

- 증상: 상주 선택 시 우측 **상세 패널(관광지·맛집 TourAPI 목록 카드)** 과 좌하단 **범례(지도 안내)** 위로 관광지·맛집 물방울 핀이 올라와 겹침.

## 2. 검증 방법 (자동화 hidden 탭 한계 우회)
- momuri/course 초기화는 `onMapReady`(map `load`) 게이트 뒤에 있어, 자동화(hidden) 탭에선 MapLibre `load`가 throttle되어 마커/타임라인이 안 그려질 수 있음.
- 우회: 로컬 정적 서버(`python -m http.server`) + Chrome에서 **전역 상태에 직접 대입 후 전역 렌더 함수 호출**로 강제 렌더.
  - course: `courses = await fetch(...); cur = 상주idx; render();` (drawMap의 `addSource`는 "Style is not done loading"로 throw하지만 타임라인 HTML은 그 전에 `#tl`에 이미 세팅됨 → 사이드바 검증 가능.)
  - momuri: `regions = await fetch(...); addLegend(); gemMarkers 생성; selectRegion(상주idx);` (`selectRegion`은 `if(map.getLayer('rbound-fill'))` 가드가 있어 안전.)
- 3건 모두 스크린샷으로 육안 확인:
  - course: 회색 사각형 사라지고 dot 1→2→3 얇은 연결선(선폭 2px 측정) ✓
  - momuri: 우측 패널 목록 행 위로 핀 안 겹침(강제 렌더 시 이 탭은 VWorld 타일까지 렌더됨) ✓, 범례 z-index 1200 > 마커 1000 ✓
- ⚠️ **실사이트 포그라운드 최종 확인 권장**: 브라우저/기기 차이 대비. workers.dev에서만 도메인 전용 키(Routo/VWorld) 정상.
